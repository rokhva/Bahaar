Quick Segmentation

Allows new users to separate their user groups (creating segments) faster, and with less possibility of gathering incorrect data.

Problem

If a website is really hard to navigate, a tremendous amount of insight can be gathered from where the greatest amount of new users drop off. The only way to get those numbers within Analytics is to create a segment (a group of users sharing a common trait or behavior). The full segment builder is too complex, and offers too many opportunities to create an incorrect segment, for new or learning users.

Approach

How could we simplify segmentation to the bare minimum? While providing enough structure that the users would have less room for error, and a greater chance of success.

Solution

The solution was an inline segment creation experience. Within this Quick segment builder, the basic structure of a segment is already set up, waiting for the user to populate it. They can simply click into a field and type to search and add. The Quick segment builder is limited to 3 rules (while the full segment builder has no maximum). In addition to the limit, there is a data indicator at the top right of the quick segment builder. Giving the users confidence that their Quick Segment does indeed contain data.

The Process

A lot of the heavy lifting for this project was around making sure that these new Quick Segment types fit seamlessly into the Analytics ecosystem. We had to make sure that the Quick segments behavior matched up with expectations as far as saving and creating workflows went.

The Result

New and learning users will now have a way to create segments in line with confidence. Quick segments can be converted into full segments. So they are both convenient and serve as a step towards learning how to make more robust segments.
